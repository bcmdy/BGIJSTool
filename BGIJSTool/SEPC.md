# BetterGI 脚本管理工具 — 需求规格说明书（v1.2）

## 1. 项目概述

### 1.1 项目名称
BetterGI 脚本管理工具

### 1.2 项目背景
用户在使用 BetterGI（Better Genshin Impact）自动化工具时，需要对脚本文件进行批量备份、删除和还原操作。本工具旨在提供一个带有图形界面的管理程序，通过读取 JSON 配置文件，实现对指定文件的自动化管理。

### 1.3 目标用户
BetterGI 脚本使用者，需要对脚本文件进行批量维护操作。

---

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 配置读取
- **FR-001**: 程序启动时自动读取 `config.json` 配置文件。
- **FR-002**: 配置文件格式为 JSON，包含以下字段：
  - `BGIpath`（字符串，必填）：BetterGI 的安装根目录，如 `E:\YS\BetterGI`。
  - `modules`（数组，原 `md`）：功能模块列表，每个模块包含：
    - `name`（字符串）：功能名称，如 `"修复稻妻大炮bug【删除】"`。
    - `files`（数组，有序）：按执行顺序排列的步骤列表，每个步骤包含：
      - `op`（字符串）：操作类型，`"bak"` = 备份，`"del"` = 删除，`"restore"` = 还原（从 backup 目录还原到目标位置）
      - `paths`（字符串数组）：当前操作对应的文件相对路径列表
    - 所有路径均为**相对路径**，相对于 `BGIpath\User\JsScript\`。

> **命名规范说明**：`files` 从对象映射改为有序数组，步骤的排列顺序即为执行顺序，用户可通过调整数组元素的顺序精细控制执行流程。

#### 2.1.2 路径解析
- **FR-003**: 文件的完整路径拼接规则：
  ```
  完整路径 = {BGIpath}\User\JsScript\{file_relative_path}
  ```
  例如：
  ```
  E:\YS\BetterGI\User\JsScript\AAA-Artifacts-Bulk-Supply\assets\ArtifactsPath\额外\所有额外\准备\000【复位程序】稻妻踏鞴砂大炮点.json
  ```

#### 2.1.3 备份功能（bak）
- **FR-004**: 将 `files.bak` 中指定的文件，按原目录结构备份到程序目录下的 `backup` 文件夹中。
- **FR-005**: 备份路径规则：
  ```
  备份路径 = {程序目录}\backup\{file_relative_path}
  ```
  例如：
  ```
  {程序目录}\backup\AAA-Artifacts-Bulk-Supply\assets\ArtifactsPath\额外\所有额外\准备\000【复位程序】稻妻踏鞴砂大炮点.json
  ```
- **FR-006**: 备份时自动创建目标目录层级（若不存在）。
- **FR-007**: 若备份目标文件已存在，覆盖原文件。
- **FR-008**: 备份完成后输出日志：`[备份成功] 源文件 -> 备份路径`
- **FR-009**: 若 `files` 中不存在 `"bak"` 键，则跳过备份步骤。

#### 2.1.4 删除功能（del）
- **FR-010**: 直接删除 `files.del` 中指定的文件。
- **FR-011**: 若文件不存在，输出警告日志：`[警告] 文件不存在，跳过删除: {文件路径}`
- **FR-012**: 删除完成后输出日志：`[删除成功] {文件路径}`
- **FR-013**: 若 `files` 中不存在 `"del"` 键，则跳过删除步骤。

#### 2.1.5 还原功能（restore）
- **FR-014**: 将 `backup` 文件夹中的备份文件，按原目录结构还原到 `op = "restore"` 指定的对应目录。
- **FR-015**: 还原路径规则：
  ```
  源路径 = {程序目录}\backup\{file_relative_path}
  目标路径 = {BGIpath}\User\JsScript\{file_relative_path}
  ```
- **FR-016**: 还原时自动创建目标目录层级（若不存在）。
- **FR-017**: 若备份源文件不存在，输出错误日志：`[错误] 备份文件不存在，无法还原: {备份路径}`
- **FR-018**: 还原完成后输出日志：`[还原成功] 备份路径 -> 目标路径`
- **FR-019**: 若 `files` 中不存在 `"restore"` 步骤，则跳过还原。

#### 2.1.6 执行顺序
- **FR-020**: 单个模块的执行顺序由 `files` 数组中元素的排列顺序决定，从前往后依次执行。
- **FR-021**: 每一步骤独立处理，互不影响。即使多个步骤操作同一文件，也按 `files` 数组中的顺序逐步骤执行。

### 2.2 UI 界面需求

#### 2.2.1 主窗口
- **UI-001**: 程序启动后显示主窗口，窗口标题为 `"BetterGI 脚本管理工具"`。
- **UI-002**: 窗口尺寸建议为 800×600 像素，支持调整大小。

#### 2.2.2 配置路径显示
- **UI-003**: 顶部显示 BetterGI.exe 的完整路径，格式如 `BetterGI.exe: E:\YS\BetterGI\BetterGI.exe`；若未找到 exe，显示红色警告并提示"请重新选择"。
- **UI-004**: 若 BGIpath 未设置或路径/文件无效，显示红色警告提示，并禁用执行按钮。

#### 2.2.3 功能选择下拉列表
- **UI-005**: 提供下拉列表（ComboBox），列出 `modules` 数组中所有模块的 `name` 字段。
- **UI-006**: 下拉列表首项为占位提示（如 `<-请选择需执行的模块->`），默认选中占位项；为空时仅显示占位项，执行按钮保持禁用。
- **UI-007**: 下拉列表宽度自适应内容，支持下拉展开。
- **UI-008**: 选择具体模块后，在日志区输出该模块将执行的操作类型及对应文件列表，并启用执行按钮。

#### 2.2.4 操作按钮
- **UI-009**: 提供 `"执行"` 按钮，点击后执行当前选中模块的所有操作（按 files 数组顺序依次执行）。
- **UI-010**: 提供 `"刷新配置"` 按钮，点击后重新读取 `config.json`。
- **UI-011**: 按钮状态：
  - BetterGI.exe 路径有效时，按钮可用。
  - 路径无效或未选择模块时，按钮禁用。
- **UI-012**: 提供 `"浏览…"` 按钮，用于手动选择 BetterGI.exe，选择后自动保存路径并刷新界面。

#### 2.2.5 日志输出框
- **UI-013**: 提供只读 RichTextBox，用于输出操作日志。
- **UI-014**: 日志框支持自动滚动，新日志追加到底部。
- **UI-015**: 日志格式：
  ```
  [HH:mm:ss] [操作类型] 详细描述
  ```
  例如：
  ```
  [14:32:15] [信息] 开始执行模块: 修复稻妻大炮bug【删除】
  [14:32:15] [信息] 操作类型: 备份(bak) + 删除(del)
  [14:32:15] [备份成功] E:\YS\BetterGI\User\JsScript\... -> backup\...
  [14:32:16] [删除成功] E:\YS\BetterGI\User\JsScript\...
  [14:32:16] [信息] 模块执行完成，共处理 4 个文件
  ```
- **UI-016**: 不同级别的日志使用不同颜色：
  - 信息：黑色
  - 成功：绿色
  - 警告：橙色
  - 错误：红色

#### 2.2.6 状态栏
- **UI-017**: 底部状态栏显示当前状态，如 `"就绪"`、`"执行中..."`、`"执行完成"`、`"配置错误"`。

## 3. 非功能需求

### 3.1 性能需求
- **NFR-001**: 程序启动时间不超过 2 秒。
- **NFR-002**: 单次操作（备份/删除/还原）响应时间不超过 1 秒（单文件）。
- **NFR-003**: 批量操作（如 100 个文件）总耗时不超过 10 秒。

### 3.2 可靠性需求
- **NFR-004**: 操作过程中发生异常时，捕获异常并输出错误日志，不导致程序崩溃。
- **NFR-005**: 还原时验证源文件存在性后再执行还原。

### 3.3 兼容性需求
- **NFR-007**: 支持 Windows 7/10/11 操作系统。
- **NFR-008**: 支持中文路径和文件名。
- **NFR-009**: 配置文件 `config.json` 使用 UTF-8 编码。

### 3.4 可维护性需求
- **NFR-010**: 代码结构清晰，采用 MVVM 或分层架构。
- **NFR-011**: 日志输出同时写入日志文件（`logs\yyyy-MM-dd.log`），便于排查问题。

## 4. 配置文件规范

### 4.1 文件格式
- 文件名：`config.json`
- 编码：UTF-8
- 格式：JSON

### 4.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `BGIpath` | string | 是 | BetterGI 安装根目录，如 `E:\YS\BetterGI` |
| `modules` | array | 是 | 功能模块列表（原 `md`） |
| `modules[].name` | string | 是 | 模块名称，用于 UI 下拉列表显示 |
| `modules[].files` | array | 是 | 有序步骤数组，排列顺序即为执行顺序 |
| `modules[].files[].op` | string | 是 | 操作类型：`"bak"` 备份 / `"del"` 删除 / `"restore"` 还原 (从 backup 目录还原到 BGIpath) |
| `modules[].files[].paths` | string[] | 是 | 当前操作对应的文件相对路径列表 |

### 4.3 路径规则
- `paths` 中的路径均为**相对路径**，相对于 `{BGIpath}\User\JsScript\`
- 路径分隔符使用双反斜杠 `\\`（JSON 转义）
- 路径分隔符使用双反斜杠 `\\`（JSON 转义）

## 7. 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| config.json 不存在 | 弹出错误提示，创建默认配置文件模板 |
| config.json 格式错误 | 弹出错误提示，显示具体 JSON 解析错误 |
| BGIpath / BetterGI.exe 路径不存在 | 路径栏红色警告，禁用执行按钮 |
| 文件操作中权限不足 | 输出错误日志，跳过当前文件继续处理后续文件 |
| 备份文件已存在 | 覆盖原文件，输出提示日志 |
| 还原时备份文件不存在 | 输出错误日志，跳过当前文件 |
| modules 为空数组 | 下拉列表仅显示占位提示项，禁用执行按钮 |
